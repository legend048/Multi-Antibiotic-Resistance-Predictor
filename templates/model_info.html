How Our AI Model Works

This system predicts antibiotic resistance for E. coli using 40 genomic features and a multi-output machine learning model. It helps clinicians and researchers estimate whether the organism is Susceptible (S), Intermediate (I), or Resistant (R) to each supported antibiotic.

What It Does

  • Predicts resistance class (S, I, R) for core antibiotics directly from genomic features.
  • Core trained antibiotics: {% for ab in antibiotics %} {{ antibiotic_info[ab].full }}{% if not loop.last %}, {% endif %} {% endfor %}
  • Extends to additional antibiotics using an online phenotype-extension model.
  • Returns confidence and class probabilities for each antibiotic.
  • Supports manual genomic input and AI-assisted extraction from medical reports.

How It Does It

  1. Collects 40 genomic features (F1-F40) from direct input or report extraction.
  2. Builds a feature vector in fixed model order.
  3. Runs a MultiOutputClassifier with one RandomForest head per antibiotic.
  4. Maps class indexes to clinical labels: S, I, and R.

How We Trained It

The model was trained on a cleaned antimicrobial resistance dataset with paired genomic features and AST outcomes. We used a multi-task setup so one training pipeline learns all {{ antibiotics|length }} core antibiotic targets together.

What Is F1 Score?

F1 score balances precision and recall into one metric, especially useful when class balance is imperfect. It ranges from 0 to 1, where higher values indicate better combined precision and recall.

F1 = 2 * (Precision * Recall) / (Precision + Recall)
